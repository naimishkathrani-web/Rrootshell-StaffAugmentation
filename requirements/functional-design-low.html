Low-Level Functional Design

Detailed specifications for each module

üîê Authentication Module

Login Functionality

  • Input: Email, Password
  • Validation: Email format, Password min 6 chars
  • Process: bcrypt password comparison
  • Output: JWT token (24hr expiry), User role, User details
  • Error Handling: Invalid credentials, Account locked

Session Management

  • Store JWT in localStorage
  • Include token in Authorization header
  • Auto-logout on token expiry
  • Refresh token mechanism

üìÑ Contract Management

Contract Creation Form

  • Fields: Contract Number (auto-generated), Client Name, Start Date, End Date, Contract Value, Currency (USD/INR/EUR/GBP/JPY/AUD/CAD/SGD), Status, Description
  • Validation: End date > Start date, Contract Value > 0, Client name required, Currency required
  • Auto-generation: Contract number format: CON-YYYY-XXX
  • Currency Support: Multi-currency support with proper symbol display ($ for USD, ‚Çπ for INR, ‚Ç¨ for EUR, ¬£ for GBP, etc.)
  • Currency Formatting: Values formatted according to currency locale (e.g., INR uses Indian numbering system with lakhs/crores)

Contract List View

  • Display: Table with Contract#, Client, Dates, Contract Value (with currency symbol), Currency, Status, Actions
  • Filters: Status (Active/Pending/Expired), Client name, Date range, Currency
  • Sort: By start date, client name, status, contract value
  • Pagination: 20 contracts per page
  • Currency Display: Contract values shown with appropriate currency symbol and formatting

üë• Resource Management

Resource Profile

  • Personal: Name, Email, Phone, Location
  • Professional: Role, Skills (multi-select), Hourly Rate, Years Experience
  • Status: Available, Assigned, On Leave
  • Current Assignment: Contract ID, Start Date, Role

Assignment Process

  1. Admin selects contract
  2. System suggests available resources matching skills
  3. Admin assigns resource
  4. Resource status updated to "Assigned"
  5. Contract updated with assigned resource
  6. Notification sent to employee

üí∞ Purchase Orders

PO Creation

  • Fields: PO Number, Contract (dropdown), Amount, Currency (USD/INR/EUR/GBP/JPY/AUD/CAD/SGD), Start Date, End Date
  • Calculation: Monthly burn rate = Amount / Months
  • Utilization: Current spent / Total amount * 100
  • Currency Display: Show currency symbol and code with amounts (e.g., "$240,000 USD", "‚Çπ5,00,00,000 INR")
  • Currency Validation: Must match associated contract currency

PO List View

  • Columns: PO Number, Contract, Client, Start Date, Expiration Date, PO Value, Currency, Utilization, Status, Actions
  • Currency Filter: Filter POs by currency type
  • Sorting: Allow sorting by any column including currency

Expiration Alerts

  • Trigger: PO end date within 30 days
  • Recipients: Admin, Contract manager
  • Frequency: Weekly email until renewed
  • Content: PO#, Client, Amount with Currency, Days remaining

‚è±Ô∏è Timesheet Module

Timesheet Grid

  • Layout: Rows = Projects, Columns = Days (Mon-Sun)
  • Input: Hours per day (0-24)
  • Totals: Daily total, Weekly total per project
  • Validation: Max 24 hours per day

Approval Workflow

  1. Employee submits timesheet (status: Pending)
  2. Notification sent to admin
  3. Admin reviews hours
  4. Admin approves/rejects with comments
  5. Notification sent to employee
  6. Status updated (Approved/Rejected)

üìÅ Document Management

Document Types

  • Contracts: MSA, SOW, NDA (linked to contract ID)
  • Tax Forms: W-9, 1099 (linked to resource ID)
  • Certifications: Professional certs (linked to resource ID)
  • Personal: Resume, references

Upload & Storage

  • Allowed types: PDF, DOC, DOCX, JPG, PNG
  • Max size: 10MB per file
  • Storage: AWS S3 or local file system
  • Metadata: Upload date, uploader, version
