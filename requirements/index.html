üìä Project Overview

Problem Statement

Currently, Rrootshell's staff augmentation operations are fragmented across multiple systems and manual processes:

  • Timesheets are submitted via email and tracked in spreadsheets
  • Contract renewals are managed manually with frequent missed deadlines
  • No centralized view of resource allocation across contracts
  • Purchase order tracking is done in separate systems
  • Reporting requires manual data consolidation from multiple sources
  • No automated notifications for critical expiration dates

This fragmentation leads to inefficiencies, errors, compliance risks, and significant administrative overhead.

Solution Approach

The Unified Timesheet & Contract Management System consolidates all operations into a single, integrated platform with three distinct user roles:

üë§ Employee Role

  • Submit timesheets (weekly/monthly)
  • Track approval status
  • View assigned projects
  • Receive notifications

üëî Admin Role

  • Manage employees & contracts
  • Configure pay/bill rates
  • Set up approval workflows
  • Generate reports

üìÅ CMG Role

  • Upload contract documents
  • Track expiration dates
  • Manage renewals
  • Version control
