üìÅ Where to Build the Real Application

All real application code goes in the repository root:

  • /app - React frontend (create using: npm create vite@latest app -- --template react-ts)
  • /server - Node.js/Express backend (create folder and initialize with npm init)
  • /database - MongoDB schemas, seeds, and migrations

Note: Do NOT build inside /mockups or /onboarding folders - those are for design review only!

üìã Important Context for All Developers

Project Vision: Rrootshell Staff Augmentation Platform is a web application for managing contracts, resources, purchase orders, timesheets, and reports for staff augmentation services.

Key Requirements: Role-based access (Admin: full access, Employee: limited to timesheets/profile/notifications), modern React UI, Node.js/Express backend, MongoDB database, deployed to GoDaddy subdomain of www.rrootshell.com

ü§ñ How to Use GitHub Copilot Prompts

  1. Open GitHub Copilot Chat in VS Code (Ctrl+Shift+I or Cmd+Shift+I)
  2. Copy the entire prompt from the dark box labeled "ü§ñ GitHub Copilot Prompt"
  3. Paste into Copilot Chat and press Enter
  4. Review the generated code - Copilot will create components, functions, and complete files
  5. Test and iterate - Ask follow-up questions like "add error handling" or "make this responsive"
